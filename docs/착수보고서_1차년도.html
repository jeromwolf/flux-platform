1.2 과업 수행 배경

본 과업은 한국해양과학기술원 부설 선박해양플랜트연구소(KRISO)의 해사 데이터를 중심으로, 산재된 해사 도메인 데이터를 지식그래프(Knowledge Graph)로 통합하고 대화형 AI 인터페이스를 통해 비전문가도 해사 데이터에 접근할 수 있는 차세대 서비스 플랫폼을 구축하는 것을 목표로 한다.

1차년도는 "설계 + PoC" 단계로, 해사 도메인 온톨로지 설계, 지식그래프 스키마 구축, ETL 파이프라인 프레임워크 개발, 그리고 GraphRAG 기반 자연어 질의응답 프로토타입 검증까지를 수행한다.

2. 연구 배경 및 필요성

2.1 해사 데이터 사일로 문제

현재 국내 해사 분야의 데이터는 다수의 기관에 분산되어 관리되고 있으며, 기관 간 데이터 상호운용성이 매우 낮은 상황이다.

기관 보유 데이터 현재 관리 방식 문제점
해양수산부 해사 정책, 규정, 공시 데이터 별도 DB + 문서 비정형 데이터 검색 난이
국립해양조사원 해도, 조석, 해류, 수심 전용 시스템 API 표준화 미흡
기상청 해양 기상, 파고, 풍속, 수온 실시간 스트림 KG 미연계
해양경찰청 해양 사고, 수색 구조, 불법 활동 내부 시스템 타 기관 데이터 연계 부재
KRISO 선박 모형 시험, 실험 결과, 연구 논문 개별 파일 시스템 메타데이터 체계 부재

핵심 문제: 해사 공무원, 어민, 해운사, 연구자 등 다양한 이해관계자가 해사 데이터에 접근하려면 각 기관의 개별 시스템을 파악하고 전문 쿼리 언어(SQL 등)를 사용해야 하며, 기관 간 데이터를 수동으로 매칭해야 한다.

2.2 지식그래프 기반 통합의 필요성

지식그래프(Knowledge Graph)는 이질적인 데이터를 엔티티(Entity)와 관계(Relationship)의 그래프 구조로 통합하여, 다음의 핵심 가치를 제공한다:

  1. 크로스도메인 연결: 선박-항만-화물-사고-기상-규정 간 관계를 명시적으로 표현
  2. 멀티홉 추론: "A 선박이 적재한 위험물 → 해당 규정 → 최근 위반 사례"와 같은 다단계 추론
  3. 자연어 접근: Text-to-Cypher 변환을 통해 비전문가도 자연어로 데이터 조회 가능
  4. 벡터 검색 통합: 그래프 구조 + 벡터 임베딩의 결합으로 의미 기반 검색 지원
  5. 온톨로지 기반 품질 관리: OWL 표준 온톨로지로 데이터 일관성 자동 검증

B.1 PhysicalEntity — 물리적 객체

물리적으로 존재하는 해사 객체. 바다 위에서 움직이거나, 항만에 고정되어 있는 실체.

B.1.1 Vessel (선박)

바다 위에서 운항하는 모든 수상 선박. 화물선, 유조선, 어선, 여객선 등을 포함.

  • 하위 유형: CargoShip(화물선), Tanker(유조선), FishingVessel(어선), PassengerShip(여객선), NavalVessel(군함/경비정), AutonomousVessel(자율운항선박)
속성명 타입 필수 설명
mmsi INTEGER ✅ 해상이동업무식별번호 (9자리, 예: 440123456)
imo INTEGER IMO 번호 (7자리, 예: 9876543)
name STRING ✅ 선박명 (예: "해운대호")
callSign STRING 호출부호 (예: "HLBQ")
vesselType STRING 선종 (CargoShip, Tanker, FishingVessel 등)

deadweight FLOAT 재화중량톤수 (DWT)
length FLOAT 전장 (m)
beam FLOAT 선폭 (m)
draft FLOAT 흘수 (m)
yearBuilt INTEGER 건조년도

currentLocation POINT 현재 위치 (위도, 경도)
