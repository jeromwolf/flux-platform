1

목적

TER-001은 설계가 실제로 동작하는지 증명한다. 단순 설계 검토가 아니라 KRISO K8s 환경에 그래프 DB를 배포하고 샘플 데이터 3종을 적재하여 시공간·관계 탐색을 시험한다. 합격 기준(관계 정확도 90%, 3-hop 탐색, 시나리오 3건)을 충족해야 한다.

2

수행 방법

PoC 환경 구축·데이터 적재

선정 그래프 DB를 KRISO K8s 플랫폼에 배포·최적화 + 샘플 데이터 3종을 DES-003 ETL로 적재 + DES-002 물리 스키마 제약 준수 확인.

시공간·관계 탐색 기능 시험

특정 선박 궤적-인접 자산 시공간 관계 추출 + 다단계 경로(Multi-hop) 탐색 + 도메인 복합 쿼리 시나리오 검증.

성능·확장성 평가

대용량 환경 응답 성능 측정 + 신규 노드/엣지 추가 시 성능 저하·확장성 한계 측정.

3

문제점 도출

3건
HIGHISSUE-01 K8s 플랫폼 접근 권한 — KRISO 제공 의존 (M+5) 미해결
현상
PoC는 KRISO K8s 인프라에 배포. 접근 권한이 M+5에 제공되지 않으면 PoC 착수 불가.
대응 방안
착수보고서에 K8s 접근 권한 요청(M+5)을 명시. 지연 대비 로컬/사내 K8s 환경에서 사전 검증 후 이식.
HIGHISSUE-02 PoC 샘플 데이터 3종 — KRISO 지정·제공 의존 (M+3) 미해결
현상
제안서는 시험시설 메타데이터·연구논문·해양기상/사고 3종을 PoC 대상으로 구체화. M+3 제공 필요.
대응 방안
제안서 명시 대로 내부 데이터 지연 시 공개 AIS 데이터(MarineTraffic API)로 대체 적용, 실 데이터 확보 시 교체.
MIDISSUE-03 DES-001~004 완료(M+5) 후 PoC 착수 — M+5~M+7 2개월 압축 미해결
현상
PoC는 모든 설계(DES)의 완료를 전제. M+5 설계 완료 후 M+7까지 2개월에 구축·시험·보고서를 끝내야 한다.
대응 방안
설계 진행 중 PoC 환경(K8s 배포·도구 셋업)을 병행 준비. DES Step별 확정분을 점진 적재하여 일정 압축에 대비.
