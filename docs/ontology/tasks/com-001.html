COM-001 · 보안 요건

일반용역 수행 시 보안 요건 — 본 사업 수행 중 취득한 지식에 대해 사업 완료 후에도 비밀 보안을 준수하고, 연구 데이터 및 관련 소프트웨어의 유출을 방지한다.

1

목적

해사 데이터에는 보안등급이 지정된 데이터가 포함될 수 있다. COM-001은 프로젝트 전 기간에 걸친 보안 준수 체계를 규정한다. 제안서 보안관리 계획(표 II-27)의 5개 보안 항목으로 대응한다.

2

수행 방법

보안각서 제출·보안관리자 지정

계약 체결 즉시 PM 및 전 참여자 보안각서 제출 + 정·부 보안관리자 지정.

개발 자산 별도 관리

개발 컴퓨터·디스크·CD·출력물을 별도 관리하여 연구 데이터·소프트웨어 유출 방지.

연구소 내 작업

연구소가 보안관리가 필요하다고 지정하는 작업은 연구소 내에서 수행.

보안등급 데이터 지침 준수

보안 등급이 지정된 해사 데이터는 연구소 보안 지침을 준수하고, 외부 처리 시 매 건 사전 승인.

3

문제점 도출

2건
MIDISSUE-01 보안각서 제출 시점 — 계약 체결 즉시 미해결
현상
제안서는 계약 체결 즉시 전 참여자 보안각서 제출을 명시. 누락 시 보안 요건 위반.
대응 방안
착수 행정 절차에 보안각서 제출을 포함. 참여자 교체 시 교체 인력 보안각서 재제출.
MIDISSUE-02 보안등급 데이터의 외부 처리 — 매 건 사전 승인 필요 미해결
현상
AIS·시험 데이터 등 보안등급 데이터는 연구소 외부 처리 시 매 건 사전 승인 필요(특수조건).
대응 방안
연구소 외부 작업이 필요한 데이터를 사전 식별하고, 승인 절차를 격주 협의에서 일괄 처리.
