1

목적

TER-002는 사람이 KG를 눈으로 탐색할 수 있는지 검증한다. 노드-엣지 그래프 시각화, 선박 항적과 지식 노드를 지도·타임라인에 결합, 대용량 렌더링 성능이 핵심이다.

2

수행 방법

자산 탐색·관계 시각화

Neo4j Bloom 또는 D3.js 기반 1-hop/2-hop 연관 자산 탐색·레이아웃 + 노드 클릭 시 속성 상세 패널 + 분류체계 계층 탐색 UI.

시공간 데이터 연계 시각화

선박 항적과 연관 지식 노드(기상·법령)를 Leaflet 지도와 결합 + 타임라인 슬라이더 연동 동적 그래프 변화.

시각화 성능·대용량 대응 측정

대규모 노드/엣지 렌더링 응답 성능 + 노드 필터링·검색·클러스터링 인터페이스 적정성 평가.

3

문제점 도출

3건
MIDISSUE-01 시각화 도구 선택 — Neo4j Bloom(상용) vs D3.js(오픈소스) 미해결
현상
Bloom은 강력하나 Enterprise 라이선스. D3.js는 무료지만 개발 공수 큼.
대응 방안
PoC는 D3.js 또는 Neo4j 무료 시각화로 검증, Bloom은 2차년도 라이선스 검토 사항으로 분리.
MIDISSUE-02 대용량 렌더링 성능 — 1,000노드 3초 목표 검증 미해결
현상
제안서 합격 기준은 1,000노드/2,000엣지 3초 이내. 웹 기반 렌더링에서 도전적인 목표.
대응 방안
노드 클러스터링·레벨오브디테일(LOD)·가상화 기법을 설계에 포함. 목표 미달 시 KRISO와 기준 협의(제안서 리스크 관리).
LOWISSUE-03 지도 라이브러리 — Leaflet vs 대안 미해결
현상
선박 항적 지도 시각화에 Leaflet을 제안. 해도(S-100) 표시 요구 시 전용 라이브러리 검토 필요.
대응 방안
PoC는 Leaflet + OSM 기본. S-100 전자해도 오버레이는 2차년도 검토.
