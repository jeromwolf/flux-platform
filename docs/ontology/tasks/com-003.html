1

목적

COM-003은 요구사항 변경의 공식 절차를 규정한다. 제안서는 격주 기술 협의(표 II-25)를 협의 채널로 운영하며, 최소 격주 1회 이상 진행 상황을 공유한다.

2

수행 방법

격주 정기 협의

격주 화요일 오전 10시 화상 또는 연구소 방문 — 상호 협의로 확정.

4-Item 보고

전주 진행 실적 · 금주 계획 · 리스크/이슈 · 연구소 확인 필요 사항.

협의록 작성·공유

매 협의 후 3일 이내 회의록 작성 → GitHub /05_reports/meeting_minutes/ 업로드.

긴급 이슈 서면 보고

일정·범위 변경 리스크 발생 시 즉시 감독관 서면 보고, 5일 이내 대응방안 제출.

3

문제점 도출

2건
MIDISSUE-01 협의 채널·주기 확정 필요 미해결
현상
제안서는 격주 화요일 오전 10시를 제안. 실제 시점·방식은 KRISO와 합의되어야 한다.
대응 방안
착수보고 회의에서 정기 협의 요일·시간·방식(화상/방문)을 1차 안건으로 확정.
LOWISSUE-02 요구사항 변경 이력 문서화 미해결
현상
상호 협의로 변경된 요구사항이 문서화되지 않으면 검수 시 범위 해석에 혼선이 생긴다.
대응 방안
변경 사항을 협의록에 기록하고, 영향받는 태스크 페이지의 문서 업데이트 로그에 반영.
