왜 이 태스크가 먼저인가

REQ-001은 모든 후속 작업의 입력이다. 제안서 스탠포드 7단계 방법론에서 Step 3 핵심 용어 열거는 명시적으로 "REQ-001 데이터 인벤토리 기반 도출"로 정의되어 있다. 즉 REQ-001 없이는 DES-001 온톨로지를 시작할 수 없다.

REQ-001의 본질은 "우리 데이터가 무엇이고, 어떻게 연결되며, 무엇을 먼저 지식그래프로 만들까"를 정하는 것이다. 결과물 기준으로 보면 산출물 6종을 만드는 일이다.

✦
막막할 때 핵심 전략: 6종을 동시에 만들지 말 것. AIS 한 개로 인벤토리·템플릿을 먼저 완성하면, 나머지 7종 데이터(GPS·RADAR·VHF·Port-MIS·CCTV·환경정보·전자해도)는 같은 틀의 복제다. 제안서도 "AIS를 대표 사례로 표준화"라고 명시했다.

해양공학수조·선형시험수조·빙해수조·심해공학수조·파력시험장·고압챔버·캐비테이션터널·선박운항시뮬레이터를 시설→시험→관측→분석→결과→문서 구조로 분석.

멀티모달 데이터 인벤토리 구축

8종 데이터를 9개 컬럼 표로 정리. 향후 온톨로지 설계와 KG 대상 선정의 기준 산출물.

AIS 중심 표준 분석 템플릿

AIS 1종을 깊게 분석하여 인벤토리 작성 기준을 표준화 → 나머지 데이터셋의 본보기.

선박(MMSI·IMO·Call Sign)·시간·위치·시설·문서·이벤트 기준의 조인 키와 관계 생성 규칙 정의.

구축 대상 자원 우선순위 선정

연결밀도 40% · 활용성 35% · 품질 25% 가중치로 평가 → Core(80↑)/Extended(60-79)/Future(60↓) 분류.

3

문제점 도출

3종 문서 정독 결과 · 6건
HIGHISSUE-01 보유 문서가 "동향 조사"라 KRISO 검수 산출물 6종은 새로 작성해야 함 진행 중
현상
docs/REQ-001_해사_KG_동향_조사.md는 글로벌 KG 이니셔티브·기술 동향·표준을 다룬 배경 리서치다. 과업지시서 REQ-001이 요구하는 것은 실제 데이터 인벤토리 구축이다.
영향
산출물 6종(현황 조사서·자원구조 분석서·인벤토리·AIS 템플릿·식별자 정의서·우선순위 선정표)을 모두 신규 작성해야 한다.
대응 방안
보유 문서 5장(한국 해양 데이터 현황)을 외부기관·시험시설 1차 자료로 재활용하되, 6종 산출물은 9컬럼 템플릿 기반으로 신규 작성. 본 워크벤치 체크리스트로 추적.
HIGHISSUE-02 시험시설 8종 데이터는 KRISO 제공 의존 — 협조 요청 지연 시 산출물 ② 불가 진행 중
현상
시험시설 자원구조 분석서(산출물 ②)는 KRISO 내부 데이터가 있어야 작성 가능. 제안서 표 II-28은 "착수 후 2주" 협조 요청을 명시.
영향
협조 요청이 늦거나 거부되면 M+2 마감(현황 분석 보고서) 전체가 지연. 리스크 등급 높음.
KRISO 우선 요청 — 최소 정보
시험시설 데이터 전체가 아니라, 산출물 ②(자원구조 분석서)의 틀·메타데이터 작성에 필요한 최소 정보를 먼저 요청한다 — 시설 8종별 ⑴ 생성 데이터 목록(데이터 카탈로그) ⑵ 데이터 형식·갱신 주기 ⑶ 메타데이터 항목 구조. 실측 데이터·대용량 파일은 후속 수신해도 무방.
대응 방안
① 착수보고서에 위 최소 정보 협조 요청을 명문화(요청 시기·사유 포함). ② 데이터 도착 전까지 외부 공개 데이터로 인벤토리 골격을 선행 작성. ③ 정상 수신 시 틀에 데이터를 채우고, 수신이 늦으면 틀을 먼저 완성·제출한 뒤 확보 시 갱신본을 재제출한다 (제안서 표 II-26·II-11 "지연 시 대체, 확보 시 교체").
MIDISSUE-03 9컬럼 데이터 인벤토리 템플릿 미확정 — 조사 시작 전 고정 필요 미해결
현상
모든 데이터를 동일한 틀로 조사해야 산출물 ⑥ 우선순위 비교가 가능. 템플릿이 흔들리면 재작업 발생.
대응 방안
아래 "9컬럼 인벤토리 템플릿"을 W1에 확정하고 KRISO 감독관과 1차 협의에서 합의.
MIDISSUE-04 외부 6기관 데이터 접근 방식 불명 (해수부·기상청·해경 등) 미해결
현상
해수부·조사원·측위정보원·기상청·해경·자율운항선박실증센터의 데이터는 공개 API/신청/비공개가 혼재. 해경 사고 데이터는 통계 수준만 공개(보유 문서 5.4 확인).
대응 방안
KRISO에 외부기관 데이터 접근 소개·협조 요청(제안서 표 II-28, 착수 후 1개월). 접근 불가 데이터는 "메타데이터 수준 조사"로 범위 한정하고 보고서에 명시.
MIDISSUE-05 멀티모달 데이터 범위 불일치 — 과업지시서 vs 제안서 미해결
현상
과업지시서는 "궤적·레이다·CCTV·VHF·전자해도·항해술·법령"을 예시. 제안서 표 II-3은 선행연구 기반 8종(AIS·GPS·센서·RADAR·VHF·Port-MIS·CCTV·환경정보)을 명세. 전자해도·항해술·법령의 인벤토리 포함 깊이가 불명확.
대응 방안
핵심 8종은 제안서 표 II-3 기준 정밀 조사, 전자해도·항해술·법령은 "확장 후보"로 메타 수준 조사. 범위를 KRISO 1차 협의에서 확정.
LOWISSUE-06 보유 문서 정보 노후화 — Activepieces·103엔티티 등 구버전 기재 미해결
현상
보유 문서가 워크플로우 엔진을 Activepieces 0.78.0, 온톨로지를 103엔티티/45관계로 기재. 현재는 Argo Workflow 전환, 온톨로지 147엔티티/108관계.
대응 방안
5장 데이터 현황만 발췌 재활용하고, 기술 스택 서술은 인용하지 않음. 신규 산출물에 구버전 정보 유입 차단.

데이터 인벤토리 템플릿 — 9개 컬럼

ISSUE-03 대응 · W1 확정
◆
각 데이터를 한 행으로, 9개 칸을 1→9 순서로 채운다. 그룹 A·B(1~6)는 사실 조사, 그룹 C(7~9)는 "그래프로 만들면?" 추론이다. 그룹 C가 단순 데이터 목록과 다른 점 — 조사하며 온톨로지 밑그림을 함께 그린다.
#컬럼답하는 질문AIS 예시
그룹 A — 이게 무슨 데이터인가
1데이터 구분무슨 데이터셋인가?AIS (선박자동식별)
2형식어떤 파일·포맷인가?바이너리 텍스트 (NMEA0183)
3세부 항목안에 무슨 필드가 있나?선박정적정보·위치·침로·속력·ETA·목적지
그룹 B — 어떻게 연결되나
4연계 항목어떤 다른 데이터와 이어지나?Port-MIS·VHF·항만시설·GPS
5주요 식별자연결할 때 키가 되는 값은?MMSI · IMO 번호 · Call Sign
6시공간 정보위치·시간이 있나?O (위경도 + 수신시각)
그룹 C — 그래프로 어떻게 만드나
7KG 후보 노드그래프의 '동그라미'로 무엇이 되나?Vessel · Voyage · Position
8KG 후보 관계동그라미끼리 잇는 '선'은?ON_VOYAGE · LOCATED_IN · COMMUNICATE_WITH
9활용 시나리오대화형 서비스 어디에 쓰나?특정 선박·시점·해역 탐색의 기준축
