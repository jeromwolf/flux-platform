DES-003 · 멀티모달 데이터 지식화 및 적재(ETL) 파이프라인 설계

1

목적

DES-003은 원천 데이터를 어떻게 KG로 흘려보낼지 설계한다. 실시간 스트리밍(AIS·센서)과 배치(법령·문서)를 병행하는 하이브리드 추출, 비정형 텍스트의 엔티티/관계 추출(NER/RE), 무결성을 보장하는 적재 전략이 핵심이다.

2

수행 방법

데이터 소스별 추출 인터페이스

연구소 내부·외부 기관 데이터의 추출 방식(API·DB 커넥터·파일) + 실시간 스트리밍·배치 하이브리드 구조.

시맨틱 변환·지식 추출 규칙

원천 데이터-온톨로지 매핑 로직 + 비정형 텍스트 NER/RE + 시공간 좌표계 통합·전처리.

그래프 적재·무결성 관리

초기 대량 적재(APOC/Admin Import) + 증분 적재(MERGE) + 트랜잭션 무결성 검증.

파이프라인 운영·모니터링

적재 상태·지연·처리량 모니터링 + 데이터 갱신 주기·자동화 워크플로우 설계.

3

문제점 도출

3건
MIDISSUE-01 비정형 텍스트(법령·문서) NER/RE — 한국어 모델 선택 미해결
현상
법령·지침에서 엔티티·관계를 추출하려면 한국어 NER 모델이 필요. KoELECTRA vs LLM 기반 파이프라인 선택 미정.
대응 방안
제안서는 KoELECTRA 또는 LLM 기반을 병기. 정확도·비용·운영성 비교 후 1차년도는 설계만, 모델 선정 권고를 포함.
MIDISSUE-02 실시간 AIS 스트리밍 — 메시지 큐 도입 범위 미해결
현상
초~수분 주기 AIS를 실시간 적재하려면 Kafka 등 메시지 큐 필요. 1차년도는 설계 용역이라 구현 범위 경계가 모호.
대응 방안
1차년도는 하이브리드 추출 "설계"까지, 실제 스트리밍 구현은 2차년도. 설계서에 범위를 명확히 구분.
LOWISSUE-03 좌표계 통합 기준 (WGS-84 / TM) 미해결
현상
데이터 소스별 좌표계가 상이(WGS-84, TM 등). 단일 기준 좌표계 선택 필요.
대응 방안
WGS-84를 KG 표준 좌표계로 채택하고 변환 규칙을 전처리 로직에 명시.
4

작업 체크리스트

자동 저장됨
  • 데이터 소스별 추출 인터페이스 설계
    API·DB커넥터·파일 + 스트리밍/배치
  • 시맨틱 변환·지식 추출 규칙
    온톨로지 매핑 + NER/RE + 좌표계 통합
  • 그래프 적재·무결성 전략
    Bulk + 증분(MERGE) + 트랜잭션
  • 파이프라인 운영·모니터링 설계
    상태·지연·처리량 + 자동화
