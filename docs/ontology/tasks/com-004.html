1

목적

COM-004는 PoC(TER-001·002) 수행에 필요한 장비·소프트웨어를 누가 확보하는지 규정한다. 업체 확보가 원칙이며, 연구소 보유 검증용 소프트웨어가 있으면 제공받을 수 있다.

2

수행 방법

PoC 장비·SW 목록 산정

PoC 수행에 필요한 장비·소프트웨어 목록을 REQ-003 그래프 DB 선정 결과와 연계하여 산정.

그래프 DB 라이선스 확보

오픈소스(Neo4j Community 등) 우선 적용 — 라이선스 비용 최소화.

연구소 보유 SW 확인

연구소가 보유한 검증용 소프트웨어가 있으면 제공 가능 여부 확인.

사후관리 라이선스 정리

본 용역 이후 사후 관리를 위해 검증에 활용한 장비·SW 라이선스를 필요 시 연구소에 제공.

3

문제점 도출

2건
MIDISSUE-01 Neo4j Community vs Enterprise 라이선스 — REQ-003과 연계 미해결
현상
PoC는 Community로 충분하나 GDS·HA 등은 Enterprise 필요. 라이선스 결정이 비용에 영향.
대응 방안
REQ-003 그래프 DB 평가 결과에 따라 1차년도는 Community로 PoC, Enterprise 전환은 2차년도 검토.
LOWISSUE-02 연구소 K8s 인프라 활용 범위 미해결
현상
PoC는 연구소 K8s 플랫폼에 배포(TER-001). 업체 장비와 연구소 인프라의 경계 설정이 필요.
대응 방안
착수 협의에서 PoC 환경(연구소 K8s)과 개발 환경(업체 장비)의 역할 분담을 확정.
