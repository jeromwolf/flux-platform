COM-005 · 산출물 관리 및 제출 (GitHub)

1

목적

COM-005는 산출물의 형상 관리 체계다. 감독관 GitHub에 모든 산출물을 등록하고, 폴더 구조·커밋 규칙·PR 리뷰 절차를 정해 변경 이력을 투명하게 관리한다. 프로젝트 전 기간에 걸쳐 상시 운영된다.

2

수행 방법

리포지토리 구조 수립

01_data_analysis · 02_ontology_design · 03_poc · 04_2nd_year_design · 05_reports 폴더 구조로 산출물 유형별 관리.

커밋 규칙 정의

[날짜][담당자] 작업내용 형식 커밋 메시지 + 주요 산출물 마일스톤 태그(v0.1, v1.0).

접근 권한 설정

감독관 계정을 Repository Owner, 수행 인력은 Contributor 권한.

PR 기반 리뷰·승인

주요 산출물 제출 전 PR로 감독관 리뷰·승인 후 병합.

3

문제점 도출

2건
MIDISSUE-01 감독관 GitHub 계정·리포지토리 — 착수 시 KRISO와 셋업 필요 미해결
현상
COM-005는 "감독관의 GitHub"를 전제. 계정·리포·권한이 없으면 산출물 관리 체계가 시작되지 않는다.
대응 방안
착수보고 회의에서 감독관 GitHub 계정 확인 + 리포지토리 생성·권한 부여를 1차 안건으로 처리.
LOWISSUE-02 보안 — 해사 데이터 민감정보 GitHub 업로드 금지 미해결
현상
제안서 보안계획은 보안등급 데이터의 외부 클라우드 업로드를 금지. GitHub도 외부 저장소.
대응 방안
산출물 "문서"만 GitHub 관리, 보안등급 원천 데이터는 연구소 내부 환경에만 보관. 디지털 파일은 암호화 압축 후 업로드.
