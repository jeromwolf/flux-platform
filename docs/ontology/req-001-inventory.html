멀티모달 해사데이터 인벤토리 (Draft)

AIS 표준 분석 템플릿(④)의 9컬럼 양식으로 멀티모달 해사데이터 10종을 정리한다. 온톨로지 설계(DES-001)와 지식그래프 스키마 설계의 기준 산출물이다.

⚠
Draft v0.2 (2026-05-20). 근거: 정성제안서 표 II-6(오재용·김혜진 2024), 현황 조사서 v0.2, AIS 표준 분석 템플릿(④). 원본: docs/REQ-001_멀티모달_데이터인벤토리_draft.md
1

개요

목적 — 멀티모달 해사데이터를 9컬럼 표준 양식으로 정리하여 온톨로지 설계(DES-001)와 지식그래프 스키마 설계의 기준 산출물로 삼는다.

대상 10종 — 선행연구 8종(정성제안서 표 II-6, 오재용·김혜진 2024: AIS·GPS·항해장비※·RADAR·VHF·Port-MIS·CCTV·환경정보) + 그 외 2종(표 II-5·과업지시서 명시: 전자해도·법령). ※ "항해장비"는 표 II-6 표기 — 과업지시서·표 II-5는 "항해술"로 표기, 원문 불일치(KRISO 확인 예정).

9컬럼 — A 데이터 자체 / B 연결 정보 / C 그래프 설계. 아래 표 1(A·B)·표 2(C)로 분할 제시.

종합 — 지식그래프 후보 노드·관계 집계

후보 노드군 (중복 제거)

  • 선박 중심 — Vessel·Voyage·Position·Event
  • 항만 중심 — Port·Berth·PortCall
  • 관측·장비 — SensorReading·Equipment·RadarTarget·CCTVDetection·WeatherObservation·ObservationStation·Camera
  • 공간 — Area·Route·NavigationArea·NavAid·Depth
  • 통신·문서 — VHFMessage·VTSCenter·Regulation·LegalReference

후보 관계군

  • 항행 — ON_VOYAGE·HAS_POSITION·LOCATED_IN
  • 입출항 — ARRIVES_AT·DEPARTS_FROM·BERTH_TO
  • 탐지·관측 — DETECTED·MATCHED_WITH·OBSERVED·MEASURED_BY
  • 통신 — TRANSMITTED_BY·COMMUNICATE_WITH·PROVIDED_BY
  • 공간 — CONTAINS·NEAR·OVERLAPS
  • 규정·영향 — APPLIES_TO·REGULATED_BY·AFFECTS·EQUIPPED_ON
✦
핵심 발견 — ① Vessel·Position·Voyage가 AIS·GPS·RADAR·CCTV·Port-MIS에 공통 등장 → KG 허브 노드. ② MMSI·호출부호·IMO가 선박 관련 데이터 6종을 연결 → 식별자 체계(⑤)의 핵심. ③ 환경정보·전자해도는 위경도·해역으로 공간 연계 → Area 노드 중심. ④ 법령은 비정형이라 NER/RE 필요(DES-003 ETL) — 후속 검토 자원.
