AIS 데이터 중심 표준 분석 템플릿 (Draft)

AIS를 대표 사례로 분석하고, 타 멀티모달 데이터셋 분석의 표준 양식을 제시한다. AIS 분석은 모든 멀티모달 데이터에 동일 적용 가능한 분석 기준 사례다.

개요

AIS(Automatic Identification System, 선박자동식별시스템)는 선박이 식별·위치·항행 정보를 VHF 대역으로 주기 송신하는 국제 표준 시스템(AIS 메시지 표준 ITU-R M.1371)이다. 해사서비스 플랫폼에서 가장 대표적이며 선박 중심 지식그래프의 핵심 연결축이다.

본 템플릿의 목적 — ① AIS 데이터 자체의 표준 분석, ② GPS·RADAR·VHF·Port-MIS·CCTV·환경정보 등을 동일한 틀로 확장 분석하는 기준 양식 제공.

AIS 지식그래프 매핑

5.1 후보 노드

노드핵심 속성AIS 항목 출처
Vessel (선박)mmsi · imoNumber · callSign · vesselName · vesselType · flag선박 식별·정적정보
Voyage (항차)departure · destination · eta운항정보
Position (위치)latitude · longitude · timestamp · sog · cog · heading위치·항행정보
Event (이벤트)eventType(입항·출항·교신) · time이벤트성 정보
Port · Route · Areaname · 좌표·범위AIS 운항정보(목적지) 참조 — Port-MIS·ENC 연계 시 구체화

5.2 후보 관계

관계의미근거
(Vessel)-[:ON_VOYAGE]->(Voyage)선박의 항차AIS 운항정보
(Voyage)-[:HAS_POSITION]->(Position)항차의 위치 기록AIS 동적정보
(Vessel)-[:LOCATED_IN]->(Area)선박이 위치한 해역위치 + 해역
(Voyage)-[:DEPARTS_FROM / ARRIVES_AT]->(Port)출발·도착 항만운항정보 + Port-MIS
(Vessel)-[:COMMUNICATE_WITH]->(Vessel)선박 간 교신VHF 연계
(Vessel)-[:TRANSMITTED_BY]->(VHFMessage)교신 발신정성제안서 표 II-7 (오재용·김혜진 2024)
⚠
후보 노드·관계명은 잠정(provisional)이다. 정성제안서 표 II-7(오재용·김혜진 2024)이 정의한 해사 특화 관계(TRANSMITTED_BY·PORT_ENTRY·PORT_DEPARTURE·ANCHOR_AT·BERTH_TO·UNBERTH_FROM·COMMUNICATE_WITH·PROVIDED_BY·LOCATED_IN)와 본 분석 제안 관계(ON_VOYAGE·HAS_POSITION)를 구분한다. 영문 명칭·방향의 최종 확정은 DES-001 온톨로지 설계 단계에서 수행한다. §5.1의 Vessel·Voyage·Position·Event는 AIS 직접 도출 노드, Port·Route·Area는 타 데이터 연계 시 구체화되는 공간 노드 — 인벤토리 ③의 AIS 행은 직접 도출 노드 4종만 싣는다. TRANSMITTED_BY·COMMUNICATE_WITH는 AIS 단독이 아닌 VHF 교신 데이터 연계 시 생성되는 관계다.
