# 데이터 간 연계 규칙 및 식별자 체계 정의서 (Draft)

## REQ-001 산출물 ⑤

> **Draft v0.2 — 2026-05-20**
> 멀티모달 해사데이터를 연결하는 식별자와 관계 생성 규칙을 정의한다.
> 근거: 정성제안서 II.2 바·사·아 및 표 II-3, 데이터 제공처별 현황 조사서 v0.2, 멀티모달 데이터 인벤토리(③)

---

## 1. 개요

지식그래프 구축의 핵심은 데이터 그 자체보다 **서로 다른 데이터 간 연결 기준**을 정의하는 데 있다. 본 정의서는 멀티모달 해사데이터에 공통·상호 참조 가능한 식별자와 연결 규칙을 도출하여, 데이터 간 단순 병렬 나열이 아닌 **시간-공간-객체-이벤트 중심의 관계 생성 규칙**을 수립한다.

---

## 2. 식별자 — 6대 연계 기준

| 기준 | 식별자 |
|------|--------|
| 선박 기준 | MMSI · IMO 번호 · 선박명 · 호출부호(Call Sign) |
| 시간 기준 | 관측시각 · 입출항시각 · 수신시각 · 시험시각 (**UTC 통일**) |
| 위치 기준 | 위도·경도 · 해역코드 · 항만코드(UN/LOCODE) · 시설위치 |
| 시설 기준 | 항만시설 ID · 시험시설 ID · 센서 ID |
| 문서 기준 | 보고서 ID · 시나리오 ID · 규정 ID |
| 이벤트 기준 | 입항 · 출항 · 교신 · 관측 · 시험 · 분석 실행 |

## 3. 통합 고유번호 목록 (국제표준 + KRISO 내부)

통합 고유번호는 선박·공간·시설 식별 코드 중심이다. 시간·이벤트·문서 기준(§2)은 코드값이 아닌 데이터 속성·기관 내부 ID이므로 §4 연결 규칙·§5 매핑에서 다룬다.

| 구분 | 식별자 | 예시값 | 출처 |
|------|--------|--------|------|
| 국제 표준 | IMO 번호 (선박 등록번호) | IMO 9876543 | 국제해사기구(IMO) |
| 국제 표준 | MMSI (해상이동업무식별번호) | 440123456 | 국제전기통신연합(ITU) |
| 국제 표준 | UN/LOCODE (항만 코드) | KR PUS (부산항) | 유엔(UNECE) |
| 국제 표준 | S-57 피처 ID (전자해도) | LIGHTS_001 | 국제수로기구(IHO) |
| ★ KRISO 내부 | 시험시설 코드 | TT(선형수조) · CVT(캐비테이션터널) | KRISO |
| ★ KRISO 내부 | 시험 데이터셋 ID | TT-2024-0312-001 | KRISO |
| ★ KRISO 내부 | 모형선 ID | MV-VLCC-2024-07 | KRISO |
| ★ KRISO 내부 | VTS 관제구역 코드 | VTS-BUSAN-01 | 해양경찰청·해양수산부 |

> ※ KRISO 내부 식별자 코드 체계는 연구소 내부 협의를 통해 착수 초기에 확정한다 (협조요청 A-1·A-2 연계).

## 4. 데이터 연결 규칙

| 규칙 | 연결 기준 | 적용 예시 |
|------|----------|----------|
| **선박 기준** | MMSI(또는 IMO·호출부호)가 같으면 동일 선박 | AIS 위치 기록 ↔ 입출항 기록 ↔ VHF 교신 기록을 같은 Vessel로 연결 |
| **시간 기준** | 모든 시각을 UTC로 통일 후 같은 시간대 데이터 연결 | 2024-01-01 09:00 UTC 위치 기록 ↔ 같은 시각 기상 관측값 |
| **공간 기준** | 위경도·해역코드·항만코드로 공간 조인 | 특정 해역 내 선박·기상·전자해도 객체 연결 |
| **시험 기준** | 시험시설 코드 + 시험 날짜 + 모형선 ID가 같으면 동일 시험 | TT + 2024-03-12 + MV-VLCC-2024-07 → 동일 시험으로 연결 |

## 5. 식별자 ↔ 데이터셋 매핑

각 식별자가 어떤 데이터셋을 연결하는지 (현황 조사서 v0.2 조인 키 + 인벤토리 ③ 기반).

| 식별자 | 연결 데이터셋 | 표준 |
|--------|-------------|------|
| **IMO 번호** | 선박원부 · 국제선박등록 · 선박검사(KOMSA) · 재난체계 AIS · 해양사고 통계 | IMO 국제 |
| **MMSI** | AIS 동적·정적 · 연안AIS 통계 · VHF · CCTV | ITU-R M.585 |
| **호출부호** | 선박입항신고 · 선박관제(VTS) · 선박운항정보 · 한중일 입출항 | ITU |
| **UN/LOCODE** | 국가·항구코드 · 차항지·목적항 필드 | UNECE |
| **ObsCode** (관측소코드) | 조석예보 · 조위실측 · 파고 · 조류 · 해양관측소 · 환경정보 | KHOA 코드체계 |
| **위도·경도 (WGS84)** | 해양오염사고 · 해양사고 통계 · 위성 격자 · 조석·파고 관측소 · 전자해도 | 공간 좌표 |
| **ENC 셀번호 / S-57 피처ID** | 전자해도 · 수심정보 · 해안선 | IHO S-57 |
| **시험시설 코드 + 데이터셋 ID** | KRISO 시험시설 8종 데이터 (A-1 수령 후) | KRISO 내부 |
| **규정 ID / 문서 ID** | 법령·규정 · 시나리오·보고서 (§2 문서 기준) | 기관 내부 |

## 6. 관계 생성 규칙

식별자 매칭으로 지식그래프 관계를 생성하는 규칙.

| 매칭 조건 | 생성 관계 |
|-----------|----------|
| 동일 MMSI/IMO | (Vessel)-[:HAS_POSITION]->(Position), (Vessel)-[:ON_VOYAGE]->(Voyage) |
| 동일 호출부호 + 항만 | (Vessel)-[:ARRIVES_AT / DEPARTS_FROM]->(Port) |
| MMSI + 교신 시각·채널 | (Vessel)-[:TRANSMITTED_BY]->(VHFMessage), (Vessel)-[:COMMUNICATE_WITH]->(Vessel) |
| 위경도·시각 근접 | (Vessel)-[:LOCATED_IN]->(Area), (WeatherObservation)-[:AFFECTS]->(Voyage) |
| 동일 ObsCode | (ObservationStation)-[:OBSERVED]->(WeatherObservation / TidalRecord) |
| 시험시설 코드 + 날짜 + 모형선 | (TestFacility)-[:CONDUCTED]->(Test)-[:USED_MODEL]->(ModelShip) |
| 해역·사고·법령 위치 매칭 | (Incident)-[:REGULATED_BY]->(Regulation), (Regulation)-[:APPLIES_TO]->(Area) |

> **관계명은 잠정(provisional)이다.** 정성제안서 표 II-7(오재용·김혜진 2024)·표 II-8 근거 관계(TRANSMITTED_BY·BERTH_TO·ARRIVES_AT·DEPARTS_FROM·COMMUNICATE_WITH·LOCATED_IN·APPLIES_TO 등)와, 본 분석에서 도출한 제안 관계(HAS_POSITION·ON_VOYAGE·OBSERVED·AFFECTS·CONDUCTED·USED_MODEL·REGULATED_BY 등)를 함께 제시한다. 영문 명칭·방향의 최종 확정은 DES-001 온톨로지 설계 단계에서 수행하며, 인벤토리 ③·템플릿 ④와 동일 명칭 체계를 사용한다.
