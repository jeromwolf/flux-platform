# Ⅱ. 용역 실시 계획서

## 1. 과업 개요

### 1.1 과업 목적
해사 분야의 다양한 멀티모달 데이터(해양관측·선박정보·전자해도·기상·항해 등)에 대한 지식그래프 모델을 설계하고, 데이터 간 의미적 관계를 체계화하여, 향후 자연어 기반 데이터 탐색 및 AI 서비스의 기반을 구축한다.

### 1.2 과업 범위
대화형 해사서비스 플랫폼의 지식그래프 모델을 설계하는 것으로, 2차년도 그래프 DB 구축 및 자연어 탐색 기능 개발이 원활히 수행될 수 있도록 확장성 있는 설계를 수행한다.

### 1.3 5개년 로드맵 내 위치
본 용역은 5개년 2단계 사업의 **1단계 1차년도**(운영 개념 정의 및 설계)에 해당한다.

## 2. 수행 체계 — 3단계 접근

| 단계 | 명칭 | 주요 내용 |
|------|------|----------|
| 1단계 | 해사 데이터 분석 및 구축 대상 선정 | 외부·내부 데이터 현황 조사, 멀티모달 데이터 분류, 구축 대상·우선순위 도출 |
| 2단계 | 해사 도메인 온톨로지 및 지식그래프 모델 설계 | 온톨로지 설계(스탠포드 7단계), 국제표준 정합성, 플랫폼 자원 관계 모델, 그래프 DB 기술 비교 |
| 3단계 | 지식그래프 모델 검증을 위한 개념 증명(PoC) | 샘플 데이터 검증, 프로토타입 구축, 자연어 탐색 검토 |

## 3. 요구사항별 수행 계획

과업지시서 15개 요구사항(공통 5 · 연구 4 · 설계 4 · 시험 2) 전체를 수행한다.

|------|------|----------|------|

| 연구 | REQ-003 | 지식그래프 DB 기술 조사 및 최적 솔루션 평가 | 1단계 |

## 4. 수행 방법

- **온톨로지 설계 방법론**: 스탠포드 7단계 방법론 적용 (목적 정의 → 재사용 → 핵심 용어 열거 → 클래스 계층 → 속성 정의 → 제약 정의 → 인스턴스 검증)
- **선행연구 활용**: KRISO 선행연구(오재용·김혜진, 2024)의 멀티모달 해사데이터 명세·관계 유형, 학술 선행연구(MAKG·PIKG) 분석 반영
- **국제표준 정합성**: IHO S-100 등 해사 데이터 표준과의 매핑 규칙 설계
- **품질 관리**: GitHub 기반 산출물 버전 관리(COM-005), 격주 기술 협의

# Ⅲ. 용역수행자 명단 및 편성표

## 1. 수행 조직

본 과업은 해사 도메인 온톨로지 설계·그래프 DB 구현·PoC 개발 3개 전문 분야를 균형 있게 수행하기 위해 **정규 인력 4명**으로 구성한다. 과업지시서 보안 요건에 따라 외부 인력 참여를 제한한다.

## 2. 참여 인력 명단 및 업무 편성표

| 역할 | 성명 | 소속·직급 | 담당 요구사항 |
|------|------|-----------|--------------|
| PM · 온톨로지 설계 책임 | 양선희 | ㈜인사이트마이닝 인공지능분석실 실장 | DES-001 · REQ-004 · DES-004 / 과제 총괄 |
| 그래프 DB 설계 | 신동욱 | ㈜인사이트마이닝 인공지능분석실 팀장 | REQ-003 · DES-002 · TER-001 · TER-002 |
| 데이터 품질 | 이부일 | ㈜인사이트마이닝 대표이사 | REQ-001 · REQ-002 · DES-003 (품질 총괄) |
| 데이터 분석 · ETL | 이재현 | ㈜인사이트마이닝 빅데이터분석실 팀장 | REQ-001 · REQ-002 · DES-003 |

- 데이터 분석·ETL 담당은 **이재현**으로 확정. 정성제안서 표기('이동훈')는 착수보고 시 정정 보고한다 (C-3).
- 투입 인월(M/M)은 〔편성표 확정 시 기재〕
- 참여자 교체 시 연구소 사전 승인 후 교체 인력 보안각서 재제출 (COM-001)
