# 지식그래프 구축 대상 자원 우선순위 선정표 (Draft)

## REQ-001 산출물 ⑥

> **Draft v0.2 — 2026-05-20**
> 멀티모달 데이터 인벤토리(③) 10종을 3축 가중치로 평가하여 구축 우선순위를 분류한다.
> 3축 점수는 5단계 정성 평가를 환산한 잠정값이며, 데이터 실수령·품질 진단 후 정량 갱신한다.
> 근거: 정성제안서 표 II-4(자산 우선순위 평가 가중치), 멀티모달 데이터 인벤토리(③), 현황 조사서 ① §10

---

## 1. 개요

모든 데이터를 일괄 그래프화하지 않고, 플랫폼 활용성과 구축 효율을 고려한 **단계적 우선순위**를 도출한다. 인벤토리 ③의 10종을 3축 가중치로 평가하여 Core/Extended/Future로 분류한다.

---

## 2. 평가 가중치 (정성제안서 표 II-4)

| 평가 축 | 가중치 | 평가 방법 |
|---------|--------|----------|
| 연결 밀도 (Centrality) | 40% | 타 데이터와의 연관 관계 — 공통 식별자 공유 데이터셋 수 기준 |
| 활용성 및 시급성 | 35% | 대화형 서비스 시나리오(사고 대응·경로 최적화 등) 참조 빈도 |
| 데이터 품질 적합성 | 25% | 메타데이터 보유·식별자 존재·결측치 비율 |

**등급 기준** — 합산 **80점↑ Core(핵심)** / **60~79 Extended(확장)** / **60점↓ Future(후속 검토)**

### 2.1 점수 산정 방법

각 축은 아래 5단계 정성 기준으로 평가한 뒤 점수대로 환산한다. 환산 근거는 현황 조사서 ① §10(조인 키 매핑·품질 진단)과 인벤토리 ③(연계 항목·식별자)이다.

| 등급 | 점수대 | 연결 밀도 (40%) | 활용성 (35%) | 품질 (25%) |
|------|--------|----------------|--------------|------------|
| 매우 높음 | 90~100 | 5종 이상 데이터와 연계 + 선박 식별자 3종↑ 공유 | 사고대응·경로·항만 全 시나리오의 기준축 | 국제표준 포맷·식별자 명확·정형 |
| 높음 | 75~89 | 3~4종 연계 + 표준 식별자 보유 | 2~3개 핵심 시나리오 | 정형 + 표준 식별자 보유 |
| 보통 | 60~74 | 2종 연계 | 1~2개 시나리오 보조 | 반정형 또는 식별자 일부 |
| 낮음 | 45~59 | 간접·단방향 연계 | 보조적 활용 | 비정형 또는 식별자 약함 |
| 매우 낮음 | ~44 | 연계 거의 없음 | 제한적 활용 | 비정형·식별자 부재 |

> 점수는 본 draft 단계의 잠정 정성 평가다. 결측치 비율·정확도 등 정량 품질 지표는 데이터 실수령 후 현황 조사서 ① §10.4 품질 진단을 갱신하여 반영한다.

## 3. 우선순위 평가표 — 10종

각 축 0~100점, 가중합 = 연결밀도×0.4 + 활용성×0.35 + 품질×0.25.

| 데이터 | 연결밀도 (40%) | 활용성 (35%) | 품질 (25%) | 가중 합산 | 등급 |
|--------|:---:|:---:|:---:|:---:|:---:|
| AIS | 95 | 95 | 88 | **93.3** | Core |
| Port-MIS | 82 | 88 | 86 | **85.1** | Core |
| 환경정보 | 78 | 84 | 82 | **81.1** | Core |
| GPS | 76 | 84 | 85 | **81.1** | Core |
| 전자해도(ENC) | 68 | 80 | 85 | **76.5** | Extended |
| RADAR | 72 | 80 | 70 | **74.3** | Extended |
| VHF | 72 | 76 | 64 | **71.4** | Extended |
| CCTV | 70 | 72 | 64 | **69.2** | Extended |
| 항해장비 ※ | 60 | 66 | 76 | **66.1** | Extended |
| 법령 | 54 | 70 | 44 | **57.1** | Future |

### 3.1 데이터별 점수 산정 근거

| 데이터 | 연결밀도 근거 | 활용성 근거 | 품질 근거 |
|--------|-------------|-----------|----------|
| AIS | 6종(GPS·RADAR·VHF·Port-MIS·CCTV·환경정보) 연계, 식별자 3종 공유 → 매우 높음(95) | 사고대응·경로·항만 全 시나리오 기준축 → 매우 높음(95) | NMEA0183 표준·MMSI 명확, 공개본 비식별·샘플 → 높음(88) |
| Port-MIS | AIS·VHF·CCTV 연계, 호출부호·UN/LOCODE 공유 → 높음(82) | 입출항·항만운영 시나리오 핵심 → 높음(88) | 정형 CSV·OpenAPI, 호출부호 명확, 대용량 공개 → 높음(86) |
| 환경정보 | AIS·Port-MIS·CCTV와 위경도·시각 연계 → 높음(78) | 사고원인·항행안전 시나리오 → 높음(84) | ObsCode 표준·정형 API → 높음(82) |
| GPS | AIS·항해장비와 위치·시각 연계 → 높음(76) | 위치 정밀도 검증 → 높음(84) | NMEA0183·RINEX 표준·정형 → 높음(85) |
| 전자해도(ENC) | 수심·해역과 공간 연계, 선박 식별자 없음 → 보통(68) | 항로안전·S-100 매핑(REQ-004) → 높음(80) | S-57 국제표준 고품질 → 높음(85) |
| RADAR | AIS·환경정보와 시공간 연계 → 보통(72) | 표적-선박 매칭 → 높음(80) | 표적 ID만·텍스트+이미지 혼합·비공개 다수 → 보통(70) |
| VHF | AIS·Port-MIS와 MMSI·호출부호 연계 → 보통(72) | 교신 분석 → 높음(76) | 음성+STT 비정형·표준화 약함 → 보통(64) |
| CCTV | AIS·Port-MIS와 표적 MMSI 연계 → 보통(70) | 영상 표적 검증 → 보통(72) | 이미지·영상 비정형·카메라ID 비표준 → 보통(64) |
| 항해장비 ※ | GPS·AIS와 선박ID 연계, 폐쇄적 → 보통(60) | 운항상태 모니터링 → 보통(66) | 정형 CSV이나 선내 데이터로 공개 제한 → 높음(76) |
| 법령 | 해역·사고·선박과 간접 연계 → 낮음(54) | 사고 시 적용 법령 탐색 → 보통(70) | 비정형 PDF/HWP·NER/RE 필요 → 매우 낮음(44) |

> ※ **"항해장비" 표기 주의** — 본 표의 "항해장비"는 정성제안서 표 II-6(오재용·김혜진 논문)의 표기를 따른 것이다. 과업지시서 및 정성제안서 표 II-5는 동일 항목을 **"항해술"**로 표기하여 원문 간 표기가 일치하지 않는다. 데이터 범위(항해 장비 센서 데이터 vs 항해술 관련 데이터)를 KRISO에 확인할 예정이다.

## 4. 등급별 분류 결과

| 등급 | 데이터 | 의미 |
|------|--------|------|
| **Core (핵심 자원)** | AIS · Port-MIS · 환경정보 · GPS | 초기 지식그래프에 반드시 포함. PoC 우선 대상 |
| **Extended (확장 자원)** | 전자해도(ENC) · RADAR · VHF · CCTV · 항해장비 | 핵심 자원과 연결되어 단계적 반영 |
| **Future (후속 검토)** | 법령 | 향후 서비스 확장 단계 — 비정형, NER/RE 필요 |

---

## 5. 연구소 시험시설 데이터 (A-1 대기)

정성제안서 자)는 "핵심 시험시설 데이터"를 Core 자원 예시로 제시한다. 시험시설 8종(해양공학수조·선형시험수조 등)은 KRISO 협조 자료(A-1) 수령 후 동일 가중치로 평가하여 본 표에 추가한다.

---

## 6. 정성제안서 예시와의 비교 · 다음 단계

### 6.1 정성제안서 자) 예시와의 차이
- 정성제안서 Core 예시: AIS·Port-MIS·GPS·RADAR·기상환경·핵심 시험시설
- 본 평가: **RADAR**가 품질 축(식별자 약함 — 표적 ID만, 텍스트+이미지)에서 감점되어 Extended(74.3)로 분류됨
- → RADAR는 데이터 확보·품질 확인 후 재평가 대상. 본 표는 잠정 평가다.

### 6.2 다음 단계
1. KRISO 시험시설 데이터(A-1) 수령 후 8종 평가 추가
2. Core 자원(AIS·Port-MIS·환경정보·GPS)을 PoC(TER-001) 우선 적재 대상으로 선정
3. 본 선정표 → 단계별 구축 로드맵(REQ-002)·DES-001 온톨로지 범위 설정의 입력
