# 2026.03.18 KRISO 미팅 정리

> KRISO 박세길 책임연구원 × Suredata Lab

## 1. 배경 및 과제 개요

- 3+2년 과제 1단계 완료 → **신개념 플랫폼 실용화 사업**으로 연계
- 연구환경마다 도구가 다르고(Matlab, 시뮬레이터, FMU/FMI 등), 서비스화가 어려운 문제
- **DevOps / MLOps (MDT-Ops)** 개념 적용
- 최종 명칭: "대화형 해사서비스 플랫폼을 위한 지식그래프 모델 설계 연구"
- 현재는 **해상교통 도메인**부터 시작, 장기적으로 해양 전체 확장 희망

## 2. 데이터 처리 파이프라인 (ETL → ELT 전환)

> "일단 올려라, 올리고 나면 우리가 처리하겠다"

| 단계 | 내용 |
|------|------|
| (1) 수집 | AIS, LTE기지국 신호, 레이더, 카메라, GPS, 라이다, ENC(전자해도), 기상, 법령 문서 등 |
| (2) 적재 | Raw 데이터 그대로 레포지토리에 올림 (ELT) |
| (3) 기본 메타데이터 추출 | 사진 EXIF, GPS 좌표, 파일 속성 등 자동 추출 |
| (4) 콘텐츠 분석 | AI 분석 → 엔티티 추출 → 지식그래프 적재 |

### 핵심 기술 방향

- OWL, RDF 기반 관계·구조 정의
- 전자해도와 온톨로지 1:1 매핑 구조 검토
- **지식의 이원화·삼원화 레이어** 필요 → 적합한 DB 선택 필요

## 3. 활용 시나리오

- "부산항에서 특정 기간 규정 위반 선박 찾아줘" → 궤적 + 지식그래프 추론
- "3월에만 통항 금지인 구역을 4월에 추론" → **시간 조건부 지식 추론**
- 항만 준설 작업 기간 정보 등 유효 조건(시간, 구역)을 리니지에 반영
- 단순 궤적 → 숨은 인사이트 발굴 (관계 기반 추론)

## 4. 워크플로우 저작 도구

### UI 구조

- **VueFlow 기반** (n8n 스타일) — 가시성 좋은 노드 편집기
- 2개의 대화창 운영:

| 대화창 | 역할 |
|--------|------|
| 전역 대화창 | 워크플로우 자동 생성, 커넥터 탐색, 플로우 피드백 |
| 노드별 대화창 | 개별 노드 설정, 파라미터 디버깅, 실행 결과 확인 |

## 5. 인프라 스택

| 영역 | 기술 |
|------|------|
| 컨테이너 | Kubernetes |
| 인증/권한 | Keycloak |
| 모니터링 | Prometheus |
| 서비스 추적 | App Mesh, Zipkin |
| GPU 분산추론 | Ray |
| DB | PostgreSQL |
| LLM | 온프레미스 로컬 (A100 수준, 오픈소스·무라이선스) |

### 응용 vs 서비스 구분

- **응용(Application)**: 1회성, 출력 결과를 가져다 사용
- **서비스(Service)**: 지속적으로 운용

## 6. 사용자 레이어 분리

| 레이어 | 대상 |
|--------|------|
| 데이터 제공처 | 해수부, 기상청, 해경, 자율운항선박실증센터, 국토지리원 등 |
| 1차 사용자 | 내부 연구자 |
| 2차 사용자 | 해양공무원, 해운사, 어민, 해양레저, 해사연구자 |

- 외부 직접 노출 최소화 → 레이어 분리
- 팀/프로젝트 단위 환경 격리
- 과도한 사용 시 경고·정지, 서비스 포털까지 고려

## 7. Suredata Lab 역할

- KRISO와 용역 형태
- **인사이트마이닝** — 지식그래프 파트 협업
- Suredata 담당:
  - 데이터 수집·데이터 마트화·재사용화
  - AI 모델 서빙 및 자산화
  - 데이터 웨어하우스 / 데이터 파이프라인

## 8. 데이터 리니지

- 자산들이 어떻게 활용되는지 이력 추적
- MIMES, 항행금지구역, 항로표지, 임시부표 등 정적+시계열 포함
- 속도 최적화 고려

## 9. 옵저버빌리티

- Kubernetes + 에이전트 시스템 로그 통합
- 전체 시스템 로그 수집 → 오류 자동 탐지
- 사용자 그룹별 모니터링, 과도 사용 경고/정지
